# 时间格式问题修复记录

## 问题

Anspire Search 的时间范围搜索返回 500 错误，参数不稳定。

## 根因分析

**错误格式**：`2025-01-01 00:00:00` (空格分隔)
**正确格式**：`2025-01-01T00:00:00` (ISO 8601，T 分隔)

## 测试验证

| 格式 | 状态 |
|------|------|
| `2025-01-01 00:00:00` | ❌ 500 |
| `2025-01-01T00:00:00` | ✅ 200 |
| `2025-01-01` | ✅ 200 |
| `1704067200` | ✅ 200 |

## 支持的格式

1. **ISO 8601** (推荐): `YYYY-MM-DDTHH:MM:SS`
2. **仅日期**: `YYYY-MM-DD`
3. **Unix 时间戳**: 秒级整数
